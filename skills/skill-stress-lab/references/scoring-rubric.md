# Stress Test Scoring Rubric (每维度 0-100)

五个维度分别独立打分（0-100），最终总分 = 五项平均值。建议保留评分说明（例如 “80 = 中风险：... ”）。

| 维度 | 关注点 | 评分参考 |
| --- | --- | --- |
| 稳定性 | 成功率、错误率 | 100：0 失败；80：≤5% 失败；60：≤15%；40：≤30%；20：>30%；0：不可运行 |
| 性能 | 平均响应、P95、队列延迟 | 100：<2s；80：2-5s；60：5-10s；40：10-20s；20：>20s；0：超时/无响应 |
| 资源消耗 | Token、CPU、外部 API | 100：最优/低成本；80：中等；60：较高；40：高；20：极高；0：导致限流或崩溃 |
| 功能一致性 | 输出质量、格式、可重复性 | 100：完全符合预期；80：轻微偏差；60：偶发错误；40：频繁偏差；20：严重错误；0：不可用 |
| 恢复能力 | 异常处理、自恢复能力 | 100：自动恢复+告警；80：一次干预；60：多次干预；40：需人工接管；20：必须重启；0：无法恢复 |

> **提示**：如需更精细，可在表格中注明具体数据点（如“失败 1/50 次 → 80 分”）。
